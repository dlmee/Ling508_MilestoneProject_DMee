Methods

Generate database

NOTE: This will take a few minutes, it's doing a lot of processing under the hood.

You will have to run this first (but only once!), before you generate a post request. Otherwise the database will be largely empty.

Currently there are five different texts that are easily accessible, they reside in this repository for ease of access. Do note, however, that it is possible that this api could adequately scrape other html files from the web, should the text be accessible in the body section. This current build can only load one text at a time. However, future iterations could easily stack texts into a growing corpus which has useful research implications for how word senses evolve as the quantity of data grows.

url = 'http://host.docker.internal:8000/Scrapes/GM_TPaC.html'

url = 'http://host.docker.internal:8000/Scrapes/GM_TPaG.html'

url = 'http://host.docker.internal:8000/Scrapes/GM_WMM.html'

url = 'http://host.docker.internal:8000/Scrapes/GM_SG.html'

url = 'http://host.docker.internal:8000/Scrapes/GM_ATBNW.html'

Endpoint: http://localhost:5000/generatedb

Request type: POST

Response: 

  {
    "msg": "success"
  }

Find the senses of a word.

Endpoint: http://localhost:5000/find_sense

Request type: POST

Request body: JSON with field "word", e.g.

Try this out here: HTML Form 

  {
    "word": "beauty"
  }

Response: JSON list of dictionaries with word form, sense, vector, and example sentences, e.g. 

{
        "examples": [
            " Fearing at last that he was rude  he turned them away; and  behold  he was in a room that was for beauty marvellous"
        ],
        "sense": 2,
        "surface": "beauty",
        "vector": [
            [
                "fearing",
                1
            ],
            [
                "behold",
                1
            ],
            [
                "marvellous",
                1
            ]
        ]
    }

Reset the database

Resets the database to empty. This will allow population of a different database.

Endpoint: http://localhost:5000/reset

Response: {"msg": "Success. The database is now empty"}
